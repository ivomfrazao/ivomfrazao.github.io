Deep-Linking Security Model★ Flagship

Identified a user-level gap in the platform's deep-linking security model. App-to-app isolation was already in place — what was missing was a check that the user opening a deeplink was the user it was created for. Because teams were using deeplink context as an SSO substitute (passing usernames to authenticate on arrival without a password), the missing check was a user impersonation path. Added bearer token validation as a second security layer alongside the existing app-level key mechanism. The service has ~20 integrations in production; the new model is in active rollout.
